Tactical RPG Prototype

Grid-based turn-by-turn combat with A* pathfinding, built from scratch in Unreal Engine 5 using C++.

EngineUnreal Engine 5
LanguageC++
ContextAcademic
FocusGameplay Systems

Overview

A prototype for a grid-based tactical RPG built as an academic project in Unreal Engine 5. The goal was to implement core tactical gameplay systems from the ground up — no pre-built plugins, pure C++ logic.

The project covers the full loop: grid generation, unit movement, turn management, and A* pathfinding. Each system was designed to be modular and extendable so new unit types or abilities could be added cleanly.

Key Features

Grid Movement System

Dynamic grid generated at runtime. Units navigate tile-by-tile with movement range calculated and highlighted in real time.

A* Pathfinding

Custom A* implementation in C++ that accounts for blocked tiles, movement cost, and unit positions on the grid.

Turn-Based Logic

Turn manager cycling between player and AI phases, handling action points, unit order, and end-of-turn conditions.

Modular Unit System

Base unit class with overridable stats and abilities — new unit types can be added without touching core logic.

Technical Challenges

A* From Scratch in C++

Implementing pathfinding without relying on existing libraries required a solid understanding of the algorithm and clean data structures. Getting it to run efficiently on large grids while accounting for dynamic obstacles was the biggest technical hurdle.

Grid-World Coordinate Precision

Translating between grid coordinates and world-space positions without floating-point drift required careful design of the grid component. Small rounding errors caused units to misalign — fixing this taught me to be precise about coordinate systems from the very start.
